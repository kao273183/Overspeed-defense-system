# 🚗 GPS 智慧測速警示系統 (扣牌防禦版)

這是一個基於 Web 技術開發的**純網頁版**車用抬頭顯示器 (HUD) 替代方案。
專為台灣駕駛設計，針對「嚴重超速扣牌」法規（超速 40km/h 以上）提供最後一道防線的語音警示。無需安裝 App，打開瀏覽器即可使用。

## ✨ 核心特色

* **🛡️ 扣牌防禦邏輯**
    * 系統預設在 **「路段限速 + 38km/h」** 時才會觸發警報。
    * **範例**：路段限速 110km/h，警報觸發點為 **148km/h**。
    * **目的**：容許正常超車或微幅超速，但確保不會觸發「嚴重超速 40 公里」的扣牌門檻。
* **🗺️ 自動路段限速 (OSM)**
    * 整合 OpenStreetMap (Overpass API)，每 15 秒自動偵測當前座標的道路速限。
    * 若地圖無資料，自動切換回手動設定值。
* **🔊 智慧語音播報**
    * 使用瀏覽器原生 `Web Speech API`。
    * 支援自定義警示語句（如：「嚴重超速！請立即煞車！」）。
    * 支援藍牙車機播放（需將手機藍牙連接至車輛）。
* **👁️ 視覺化強烈警示**
    * 正常行駛時顯示綠色時速。
    * 觸發警報時，全螢幕**紅底閃爍**，利用眼角餘光即可察覺危險。
* **🔋 螢幕恆亮 (Wake Lock)**
    * 透過 `Screen Wake Lock API` 防止手機在行駛中自動鎖定黑屏。
* **🛑 暫停監控模式**
    * 進休息站或塞車時可一鍵暫停，停止 GPS 運作以省電。

## 🛠️ 技術棧

* **Frontend**: HTML5, CSS3 (Flexbox, Animation)
* **Scripting**: JavaScript (ES6+)
* **Web APIs**:
    * `Geolocation API` (高精確度定位)
    * `SpeechSynthesis API` (文字轉語音)
    * `Screen Wake Lock API` (螢幕恆亮)
    * `Fetch API` (非同步請求地圖資料)
* **Data Source**: OpenStreetMap (Overpass API)

## 🚀 如何使用 (Installation & Usage)

由於瀏覽器安全性限制，Geolocation (GPS) 功能**必須在 HTTPS 環境下**才能運作。

### 方法一：直接部署 (推薦)
將 `index.html` 上傳至 **GitHub Pages**、**Vercel** 或任何支援 HTTPS 的靜態網頁託管服務。

### 方法二：本機開發 (Localhost)
如果你有安裝 VS Code 與 Live Server：
1. 下載此專案。
2. 用 VS Code 開啟資料夾。
3. 點擊右下角的 `Go Live`。
4. 手機連線至電腦的 IP (例如 `https://192.168.1.X:5500`)。
   * *注意：手機連線 Localhost 可能需要設定 SSL 憑證，建議使用方法一最簡單。*

### 操作步驟
1. 開啟網頁，允許瀏覽器存取 **位置權限**。
2. 點擊「**🔊 測試語音**」確認手機音量或車機連線正常。
3. 確認「警報觸發點」數值是否符合預期。
4. 點擊「**🚀 啟動**」開始監控。
5. 將手機橫放或直放於儀表板上方固定。

## ⚠️ 注意事項與限制

1.  **GPS 訊號**：
    * 進入隧道、地下道或室內停車場時，GPS 訊號會中斷，速度數值可能會卡住或歸零。
    * GPS 速度更新頻率約為 1Hz (每秒一次)，相較於車輛儀表板會有約 **1~2 秒的延遲**，請預留反應時間。
2.  **地圖資料來源**：
    * 速限資料來自 OpenStreetMap (OSM) 開源社群。
    * 部分新開通道路、偏僻巷弄可能無速限資料，此時系統會顯示 `--` 並沿用上一次的速限設定。
    * 自動限速功能需要**網路連線**。
3.  **瀏覽器支援**：
    * 建議使用 **Chrome (Android)** 或 **Safari (iOS)** 以獲得最佳體驗。
    * iOS 用戶請注意：若開啟「省電模式」，螢幕恆亮功能可能會失效。
